What This Index Is Based On

The composite score draws on six kinds of evidence: multilingual corpora, multilingual language models, multilingual benchmarks, parallel (translation) corpora, and counts of grammars and dictionaries surveyed from Glottolog's References collection. How the Composite Score Is Calculated

For every language, a composite score combines the value of each resource dimension. For Corpus, Model, and Benchmark coverage, maximum scores represent the number of sources included. For Grammars and Dictionaries, an artificial maximum was selected in order to cap values. This is because for some languages, hundreds of grammars and dictionaries are listed, and this cap stops the inflation of scores for these instances while ensuring genuinely high-resource languages don't receive comparably low scores. For Parallel Data coverage, scores already arrive normalized on a 0–100 scale and so are rescaled linearly rather than log-compressed.

Step 1 — Normalizing Each Dimension

Each dimension's raw count cr is converted into a normalized score nr (between 0 and 1). Count-based dimensions are log-compressed against a fixed maximum Mr; Parallel Data, already a 0–100 score, is rescaled linearly instead:

nr = min ( 1, ln(1 + cr) / ln(1 + Mr) )   for r ∈ {Grammars, Dictionaries, Corpora, Models, Benchmarks}
nr = min ( 1, cr / 100 )   for r = Parallel Data

Log-normalization compresses the count scale so that, for example, the difference between 5 and 10 grammars matters more than the difference between 200 and 205. The min(1, ·) cap ensures no single dimension can contribute more than its full share regardless of how far a raw count exceeds the maximum.

Step 2 — Combining Dimensions into a Composite Score

The six normalized dimension scores are combined into a single composite score S (between 0 and 100) using a weighted average, where each dimension's weight wr determines how much influence it has relative to the others:

S = 100 · Σr∈R (wr · nr) / Σr∈R wr

where R = {Grammars, Dictionaries, Corpora, Models, Benchmarks, Parallel Data}. Because the denominator is always the sum of whichever weights are currently set, the weights don't need to add up to any particular total. This is exactly what the sliders on the Explorer tab control.

Where the Parallel Data Score Comes From

Parallel Data's input score (the 0–100 value used above) is itself computed upstream from OPUS corpus coverage, before it ever reaches the composite formula. For each language i, a score is given based on how many corpora include it, how many distinct translation partner languages it shares data with, and how many total sentence pairs exist for it overall:

score = 100 · ( wc·ĉi + wp·p̂i + ws·ŝi )

where ci is the number of corpora covering language i, pi is the number of distinct translation partners, and si is the total sentence pairs across all corpora. Each component x̂i is min–max normalized after log-compression, relative to every other language j in the dataset:

x̂i = ( ln(1+xi) − minj ln(1+xj) ) / ( maxj ln(1+xj) − minj ln(1+xj) )
